# Посібник користувача — Діагностична анкета

## 1. Про інструмент

Діагностична анкета — це screening-інструмент для первинної оцінки кандидатів. Він вимірює шість профільних осей і присвоює один із семи архетипів.

> **Важливо:** Це screening-інструмент, **не остаточний вердикт**. Результати потребують інтерпретації фахівця і не замінюють живу співбесіду.

### Шість профільних осей (0–100)

| Вісь                | Що вимірює                                                          |
| ------------------- | ------------------------------------------------------------------- |
| Когнітивний проксі  | Логічне мислення, аналіз патернів, швидкість обробки інформації     |
| Лідерство           | Командні якості, рішення у групових ситуаціях, досвід керівництва   |
| Відповідальність    | Надійність, виконавська дисципліна, відповідальна поведінка         |
| Ініціативність      | Проактивна позиція, готовність діяти без прямої вказівки            |
| Навченість          | Відкритість до нового, адаптивність, готовність до навчання         |
| Технічна готовність | Декларовані кваліфікації, верифіковані знаннями тестовими питаннями |

Інтерпретація балів:

| Бал   | Рівень                   |
| ----- | ------------------------ |
| < 35  | Слабкий — потребує уваги |
| 35–54 | Середній                 |
| 55–69 | Добрий                   |
| 70+   | Сильний                  |

## 2. Рекомендований сценарій проведення асесменту

### Дистанційне проходження

1. **Підготовка.** Згенеруйте анонімний код для кандидата. Код — довільний унікальний рядок (наприклад: `UA-2026-001`, `batch3-042`). Ніякого ПІБ.

2. **Відправлення.** Надішліть кандидату:
   - Посилання на анкету: `https://your-deployment-url/survey`
   - Його індивідуальний код
   - Пояснення: анкета займає ~30–40 хвилин, відповіді анонімні, після завершення потрібно завантажити файл результатів.

3. **Проходження.** Кандидат:
   - Вводить код на початку анкети
   - Проходить 6 блоків (базові дані, верифікація, когнітивний тест, психометричний, сценарії)
   - На сторінці результатів натискає **"Завантажити JSON"** або **"Завантажити PDF"**
   - Надсилає файл результатів вам

4. **Аналіз.** Ви отримуєте файл з повним профілем. Дивіться розділ 3 для інтерпретації.

> **Увага:** Дані зберігаються тільки в пам'яті браузера. Якщо кандидат закрив вкладку до завантаження — сесія втрачена і потрібно починати знову.

## 3. Як читати звіт

### Архетипи

| Архетип                       | Умови присвоєння                                                | Підходящі ролі                                  |
| ----------------------------- | --------------------------------------------------------------- | ----------------------------------------------- |
| **Потенційний лідер**         | Лідерство ≥ 70, відповідальність ≥ 65, ініціативність ≥ 60      | Командні посади, управлінські ролі              |
| **Технічний виконавець**      | Технічна готовність ≥ 60, відповідальність ≥ 60, лідерство < 60 | Технічні фахівці, спеціалізовані ролі           |
| **Адміністратор-координатор** | Відповідальність ≥ 65, навченість ≥ 55, лідерство 40–65         | Організаційні ролі, координатори                |
| **Універсал з потенціалом**   | Всі осі ≥ 55, різниця між максимумом і мінімумом < 25           | Адаптивні, різнопрофільні ролі                  |
| **Виконавець базового рівня** | Адекватний профіль без яскравих strengths                       | Виконавські ролі зі структурованим керівництвом |
| **Не підходить**              | ≥ 3 осей < 35                                                   | Не відповідає мінімальним вимогам               |
| **Дані недостовірні**         | Недостовірна валідність + ≥ 4 сильних осі                       | Рекомендується повторне проходження             |

### Показники валідності

| Статус           | Що означає                                                                     |
| ---------------- | ------------------------------------------------------------------------------ |
| **Надійно**      | Відповіді узгоджені, без ознак соціальної бажаності чи поспіху.                |
| **Сумнівно**     | Є певні ознаки недостовірності. Бажана жива співбесіда.                        |
| **Недостовірно** | Результати не можна вважати достовірними. Рекомендується повторне проходження. |

Окремі попередження:

- **Шкала брехні > 30%** — соціально бажані відповіді на питання-пастки
- **Узгодженість > 30%** — суперечливі відповіді на схожі питання
- **Перевірка уважності < 100%** — невірні відповіді на очевидні запитання
- **Ознаки поспіху** — дуже швидке або одноманітне заповнення

### Технічні кваліфікації

| Статус                  | Що означає                                           |
| ----------------------- | ---------------------------------------------------- |
| Верифіковано            | ≥ 66% правильних відповідей на перевірочні запитання |
| Часткова верифікація    | 33–66% правильних відповідей                         |
| Не верифіковано         | < 33% — ймовірно неправдива декларація               |
| Без перевірочних питань | Верифікаційний блок пропущено                        |

> Технічні кваліфікації з будь-яким статусом, крім "Верифіковано", потребують перевірки фізичних документів.

## 4. Сіра зона — коли потрібна жива співбесіда

Наступні ситуації **завжди потребують додаткової оцінки**:

- Валідність: "Сумнівно" або "Недостовірно"
- Архетип: "Виконавець базового рівня" або "Не підходить"
- 1–2 осі < 35 (навіть якщо загальний архетип позитивний)
- Задекларовані технічні кваліфікації зі статусом "Не верифіковано"
- Будь-яке попередження валідності для критично важливої ролі

Приблизно 40–50% кандидатів потрапляють у сіру зону при першому скринінгу — це нормально.

## 5. Обмеження інструменту

1. **Screening, не діагностика.** Анкета оцінює поведінкові патерни та когнітивний потенціал, але не є клінічним психологічним тестом.

2. **Калібровані пороги.** Пороги архетипів і ваги осей задані емпірично і потребують уточнення після пілотного запуску. До калібрування — результати орієнтовні.

3. **Документи обов'язкові.** Верифікаційний блок перевіряє знання, але не замінює фізичну перевірку посвідчень, сертифікатів тощо.

4. **Немає медичної оцінки.** Інструмент не містить клінічних шкал і не призначений для медичного або психіатричного скринінгу.

5. **Тільки браузерна пам'ять.** Дані не зберігаються на сервері. Після закриття вкладки — втрачені назавжди.

## 6. Часті питання

**Чи зберігаються дані на сервері?**
Ні. Всі дані — тільки в пам'яті браузера під час сесії. Обов'язково завантажте JSON або PDF до закриття вкладки.

**Що робити якщо архетип "Дані недостовірні"?**
Запропонуйте повторне проходження в контрольованих умовах. Причини: поспіх, нерозуміння інструкцій, навмисна фальсифікація.

**Скільки часу займає анкета?**
30–40 хвилин при уважному проходженні. Менше 20 хвилин — ознака поспіху (система фіксує це).
